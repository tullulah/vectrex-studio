# Vectrex Studio v0.1.1 — Math, MOVE, and 16-segment circles

**Date:** 25 February 2026

This release focuses on correctness: fixing math builtins that looked implemented but produced wrong results, unifying DRAW_CIRCLE to 16 segments across both compilers, and making MOVE actually work. It also ships 11 ready-to-open snippet projects as learning material.

---

## What's new

### MOVE now works in both compilers

`MOVE(x, y)` sets a coordinate origin so that all subsequent `DRAW_LINE` calls are relative to that position. It was broken in two different ways:

**Core (`vectrexc`):** `MOVE` was simply not in the builtins list. When the compiler saw `MOVE(...)`, it fell through to user-function handling and emitted `JSR MOVE` — which then failed at link time with *"undefined symbol"*.

**Buildtools (`vpy_cli`):** `MOVE` was in the list, but was calling `Moveto_d_7F` without the required BIOS setup (`DP=$D0`). Worse, `DRAW_LINE_WRAPPER` always calls `Reset0Ref` first, which resets the beam to center and cancels any prior Moveto_d call anyway.

**The fix (both compilers):** `MOVE(x, y)` now stores the coordinates into two 1-byte signed RAM variables `VPY_MOVE_X` and `VPY_MOVE_Y`. These are initialized to 0 at startup. `DRAW_LINE_WRAPPER` adds them via `ADDA/ADDB` to the start-point before the `Moveto_d` call.

```python
# draw_move example
def loop():
    MOVE(-60, 60)          # shift origin to top-left area
    DRAW_LINE(0, 0, 0, -40, 127)   # draws relative to (-60, 60)
    DRAW_LINE(0, -40, 30, -40, 127)
```

### DRAW_CIRCLE upgraded to 16 segments everywhere

Previously, the **runtime path** (when radius is a variable) was drawing an 8-segment octagon. The **constant path** was already using a 16-segment polygon. This made circles look noticeably different depending on whether the radius was a literal or a variable.

Both paths now use a 16-sided polygon. The runtime path computes the 16 vertex offsets using the new `MUL` instruction with 4 precomputed fractions:

| Fraction | Value | Approx |
|----------|-------|--------|
| a = cos(π/8)·r | (#98 × r) >> 8 | 0.3827r |
| b = cos(3π/8)·r | (#83 × r) >> 8 | 0.3244r |
| c = sin(π/8)·r | (#56 × r) >> 8 | 0.2168r |  
| d = sin(3π/8)·r | (#19 × r) >> 8 | 0.0761r |

This required adding `MUL` (opcode `0x3D`) to both assemblers — it was missing from both.

### Math builtins fixed in core

`abs()` was the only math function that actually worked. `min()`, `max()`, and `clamp()` all compiled but produced wrong results at runtime. There were four layered bugs:

**Bug 1 — Semantic gate:** `BUILTIN_ARITIES` (the list the validator checks) only had `ABS`. Calling `min(30, 70)` immediately produced *"Unknown function: min"* before code generation even started.

**Bug 2 — RAM not allocated:** `analyze_runtime_usage()` is what triggers allocation of temp RAM slots like `TMPLEFT`/`TMPRIGHT`. It never scanned for `MIN`/`MAX`/`CLAMP` calls. The assembler then failed with *"undefined symbol: TMPLEFT"*.

**Bug 3 — Wrong comparison instruction:** `MIN` was using `SUBD TMPRIGHT; BGT` — but `SUBD` modifies the D register. After the subtraction, D no longer holds the first argument. The branch target tried to store the corrupted D as the result. Replaced with `CMPD RESULT; BLE` which is non-destructive.

**Bug 4 — Assembler drops inline label instructions:** The generated ASM had lines like `MIN_FIRST_1: STD RESULT` (label and instruction on the same line). The assembler's `parse_and_emit_instruction()` receives the full raw line, splits on whitespace, and gets mnemonic = `"MIN_FIRST_1:"` — no match in the dispatch table, instruction silently dropped. `RESULT` was never written. Fixed by emitting the label on its own line followed by the instruction.

After all four fixes, `min(30, 70) = 30`, `max(30, 70) = 70`, `clamp(150, 0, 100) = 100`. ✓

### 11 snippet projects

`examples/individual_tests/` now contains 11 self-contained `.vpyproj` files that can be opened directly in Vectrex Studio. Each tests one feature:

```
draw_circle/      — DRAW_CIRCLE with various radii
draw_line/        — DRAW_LINE basic usage
draw_move/        — MOVE + DRAW_LINE offset
draw_rect/        — DRAW_RECT bounding boxes
draw_vector/      — DRAW_VECTOR asset rendering
joystick_buttons/ — J1_BUTTON_1–4 with debounce
joystick_position/— J1_X/J1_Y analog and digital
math_functions/   — abs, min, max, clamp
play_music/       — PLAY_MUSIC / STOP_MUSIC
print_number/     — PRINT_NUMBER signed decimal
print_text/       — PRINT_TEXT positioning
```

These are designed as copy/paste starting points. Open any of them in the IDE, hit **Build**, and they run in the emulator immediately.
